This language is a custom, imperative, procedural programming language.

It supports basic programming constructs including:

1. Identifiers:
   Identifiers must begin with an uppercase letter (A-Z),
   followed by lowercase letters, digits, or underscores,
   with a maximum length of 31 characters.

2. Literals:
   - Integer literals (e.g., 42, -10, +99)
   - Floating-point literals (e.g., 3.14, -0.25, 1.5e3)
   - String literals with escape sequences
   - Boolean literals (true, false)

3. Comments:
   - Single-line comments start with ##

4. Whitespace:
   Spaces, tabs, and newlines are allowed and ignored
   except inside string literals.

The language follows the longest-match principle
and pattern matching priority as defined in the
scanner specification.

This grammar defines the lexical structure of the
language and is used by the lexical analyzer to
tokenize source programs.
